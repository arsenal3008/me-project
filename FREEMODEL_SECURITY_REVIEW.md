# FreeModel.dev Manager — Обзор безопасности и улучшений

Проведён по тому же принципу, что и **AgentChecker 3.0**: разбор по
уровням критичности, исправления и новые возможности.

## 🔴 Критично

### 1. Учётные данные хранились в открытом виде без ограничения доступа
`fm_accounts.json` содержит **пароли, cookie сессий и API-ключи** в plaintext.
Любой процесс/пользователь с доступом к папке мог их прочитать.

**Сделано:**
- При сохранении файл получает права `0600` (только владелец) — `settings.secure_chmod()`.
- `fm_accounts.json` добавлен в `.gitignore` — секреты больше не попадут в репозиторий по ошибке.

**Рекомендация на будущее:** хранить секреты в ОС-хранилище (Windows Credential
Manager / `keyring`) или шифровать мастер-паролем. Сейчас сделан минимально
инвазивный шаг, не ломающий совместимость.

### 2. Приложение не запускалось вне Windows (краш при импорте)
`browser_login.py` импортировал `ctypes.wintypes` на верхнем уровне модуля.
На Linux/macOS этого модуля нет → `ImportError` при старте всего приложения
(т.к. `dialogs` импортирует `browser_login`). Также мешало автотестам.

**Сделано:** `ctypes`/`ctypes.wintypes` импортируются лениво внутри
DPAPI-функций под защитой `sys.platform.startswith("win")`. На Windows
поведение не изменилось; на других ОС приложение запускается, а функции
DPAPI просто возвращают `None`.

## 🟠 Средне

### 3. Аккаунт помечался «valid» даже при полном провале запросов
`fetch_stats()` всегда выставлял `status="valid"`, даже если все три эндпоинта
вернули ошибку (истёкшие cookie, 401/403, таймаут). Пользователь видел «OK»
у нерабочего аккаунта.

**Сделано:**
- `valid` ставится только если **хотя бы один** эндпоинт вернул данные.
- `401/403` → `status="invalid"` с пояснением «Cookie истекли».
- Прочие сбои → `status="error"` с текстом последней ошибки.
- При неудаче cookie теперь корректно срабатывает **запасной путь через API-ключ**.

### 4. Тихое проглатывание ошибок
Везде стоял `except Exception: pass/continue`, скрывавший причину сбоя.
В `fetch_stats` исключения теперь разделены (таймаут / сеть / формат) и
сохраняются в `account.error` для показа пользователю.

## 🟡 Минорно / улучшения качества

### 5. Таймеры показывали только часы и минуты
`_time_until` не показывал дни — для 7-дневного окна это неинформативно.

**Сделано:** новый формат **«дней + часов»** (`5д 12ч`, для <1 дня `3ч 45м`),
плюс абсолютное время окончания. Введён `_normalize_ts()` — корректно
обрабатывает метки в секундах **и** миллисекундах.

## ✨ Новые возможности

- **Колонки «Сброс 5ч» и «Сброс 7д»** в главной таблице — показывают,
  сколько осталось до сброса лимитов, в формате «дней + часов».
- **Живой обратный отсчёт:** значения тикают раз в минуту без перерисовки
  всей таблицы (сохраняются выделение и сортировка).
- **Авто-миграция настроек:** новые колонки автоматически появляются у тех,
  кто уже пользовался прежней версией (одноразово, без дублей).
- Расширенный блок «5ч/7д» в окне редактирования: время до сброса + дата окончания.
