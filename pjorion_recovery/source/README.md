# Восстановленный исходник mod_ZJ_ContourLook

Исходник восстановлен из защищённого PjOrion `.pyc` за **4 слоя**:
1. Снят внешний упаковщик PjOrion (zlib + побитовый реверс + marshal).
2. Снят рантайм-дамп **расшифрованного** байткода прямо в игре (`mod_zzz_pjdump`) —
   получено **130 code-объектов** (из них 74 уникальных принадлежат самому моду).
3. **Деобфускация потока управления** PjOrion: реальные инструкции были разбросаны и
   связаны цепочками безусловных переходов с мусорными байтами между ними. Написан
   деобфускатор, который проходит реальный поток, выбрасывает мусор и пересобирает
   чистый байткод.
4. Декомпиляция чистого байткода (pycdc + uncompyle6).

## Что внутри
- `ZJ_ContourLook_recovered.py` — **сводный восстановленный исходник** (классы + функции).
- `per_object/` — по одному файлу на каждый из 74 объектов мода (надёжнее для чтения по частям).
- `disasm/` — чистый дизассемблер всех 74 объектов (эталон для проверки/правки артефактов).

## Покрытие
- **70 из 74** объектов восстановлены в читаемый исходник (69 через pycdc, 1 через uncompyle6).
- **4** объекта (`status`, `GetVehicleViewAngle`, `query`, `serv_query`) — только дизассемблер
  (декомпиляторы не справились со структурой try/except); логика читается в `disasm/`.

## Что восстановилось полностью
- **Все имена** функций, методов, классов, переменных (`DrawControl`, `addEdge`,
  `removeEdge`, `ZJ_ContourLook`, `License`, `Analytics`, `ServerCheck` и т.д.).
- **Все строки** (включая русские сообщения, URL, тексты лицензии) — расшифрованы.
- **Вся логика** работы мода: отрисовка контуров, обработка боя, проверка лицензии/сервера.

## Известные артефакты декомпиляторов (важно)
Это **не до конца компилируемый** исходник — есть типичные артефакты pycdc, которые
надо поправить руками (эталон — `disasm/`):
- Потерянная переменная в цепочках `if/elif` иногда печатается как `None`
  (напр. `if None == 'lng':` — на деле `if type == 'lng':`).
- `None.databaseID` / `None.target()` — на деле там `player.databaseID` / `BigWorld.target()`
  (потерян первый `LOAD_GLOBAL` в выражении).
- Импорты вида `from X import Y` иногда печатаются как `Y = Y` + `import X`.
- Заголовок создания класса печатается как `class MyServerGO(None, None, ...)`.
- Иногда появляются лишние `continue` / `return None` на краях блоков.

Логика везде верна — артефакты косметические и локальные. Для точной сверки любого
места используйте соответствующий файл в `disasm/`.
